# 群组管理功能使用说明

## 功能概述

机器人现在支持完整的群组管理功能，包括：
- 自动检测和删除违规内容（诈骗、色情、垃圾信息）
- 用户管理（封禁、踢出、警告、禁言）
- 违规记录和统计

## 自动功能

### 关键词过滤
机器人会自动检测以下类型的违规内容：

**诈骗关键词：**
- 刷单、兼职、投资理财、贷款、中奖等

**色情关键词：**
- 色情、成人、约炮、特殊服务等

**垃圾信息关键词：**
- 广告、推广、加群、转发等

### 自动处理规则

1. **诈骗信息**：删除消息 + 警告（3次警告后自动封禁）
2. **色情内容**：删除消息 + 立即封禁
3. **垃圾信息**：删除消息 + 警告（5次警告后自动封禁）

## 管理员命令

### 基础命令

#### `/ban` - 封禁用户
```
/ban [回复用户消息]
/ban [回复用户消息] 原因说明
```
封禁用户，用户将无法重新加入群组。

#### `/unban` - 解封用户
```
/unban [回复用户消息]
```
解封之前被封禁的用户。

#### `/kick` - 踢出用户
```
/kick [回复用户消息]
/kick [回复用户消息] 原因说明
```
踢出用户，用户可以重新加入群组。

#### `/warn` - 警告用户
```
/warn [回复用户消息]
/warn [回复用户消息] 原因说明
```
警告用户，系统会记录警告次数。

#### `/mute` - 禁言用户
```
/mute [回复用户消息]
/mute [回复用户消息] [小时数]
```
禁言用户，默认1小时，最长168小时（7天）。

#### `/del` - 删除消息
```
/del [回复要删除的消息]
```
删除指定的消息。

#### `/stats` - 违规统计
```
/stats
/stats [天数]
```
查看违规统计信息，默认显示最近7天。

## 使用要求

1. **机器人权限**：机器人必须是群组管理员，并且拥有以下权限：
   - 删除消息
   - 封禁用户
   - 限制用户

2. **管理员权限**：只有群组管理员可以使用管理命令。

## 数据库

系统会自动创建 `group_admin.db` 数据库文件，用于存储：
- 违规记录
- 封禁用户列表
- 警告记录

## 自定义关键词

推荐通过 Telegram 管理命令在「当前群/频道」内动态调整（无需重启，且会持久化到 `group_admin.db`）：

- `/kwadd 关键词`：添加自定义屏蔽词（仅本群/频道生效）
- `/kwdel 关键词`：删除自定义屏蔽词（仅本群/频道生效）
- `/kwlist`：查看当前自定义屏蔽词列表

说明：
1. 仅群组管理员可用
2. 匹配不区分大小写
3. 默认内置关键词始终生效；`/kwlist` 只展示“自定义”部分

## 注意事项

1. 确保机器人有足够的管理员权限
2. 关键词匹配不区分大小写
3. 违规记录会永久保存到数据库
4. 封禁的用户需要手动解封才能重新加入

## 故障排除

如果命令不工作：
1. 检查机器人是否为管理员
2. 检查机器人是否有相应权限
3. 检查您是否为群组管理员
4. 查看日志文件获取详细错误信息

如果出现 Gemini SDK 相关报错（例如 `has no attribute 'GenerativeModel'`）：
1. 确认运行环境同时安装了 `google-genai`（新 SDK）与 `google-generativeai`（旧 SDK，作为回退）
2. 确认环境变量 `GEMINI_API_KEY` 已正确设置
